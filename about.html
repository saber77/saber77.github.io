Project Overview

Video game is an electronic game that can be played on a computing device, such as a personal computer, gaming console or mobile phone. Video games have come a long way since the first games emerged in the 1970s. Todayâ€™s video games offer photorealistic graphics and simulate reality to a degree which is astonishing in many cases such as Ghost of Tsushima, Flight Simulator. Data from 2021 shows that the number of gamers around the globe are estimated to be around 3 Billion including computer games, mobile games and console games.

Given the huge market size it is interesting to try to understand how the video game industry has evolved and what insights we can extract from studying video game data. The project is done by Alankrit Gupta and focuses mainly on Steam PC games Steam. It uses data hosted on mendeley and data.world. The aim for the project is exploratory and will focus on insight generation while testing out capabilities of python library Altair.

Design Methodology

Initial Assessment

Being a video game enthusiast I was very comfortable with the data domain and had a solid understanding fo what sort of trends I should be expecting. The game details dataset was pretty standard with almost all the information one wold expect from sales and description data. I had an exhaustive list of questions mostly pertaining to active player count, what kind of games get rated highly, video game description analyis and finally how gaming has become more demanding over the years. The part that was the most exciting was going to be how can I make simple charts I had in mind interactive and get them to talk to each other. As part of requirement gathering I identified the main topics and the deep dive questions related to each topic as under:

The main topics that were to be explored:

  1. Player Retention
  2. Video Game Scores
  3. Video Game Description
  4. Video Game Requirements

The deepdive questions associated with the topics:

  1. How are player counts growing/declining over years ?
  2. How do single player games compete against multiplayer games ?
  3. Do we see any particular outliers in active player counts ?
  4. What genre of games are highly rated ?
  5. What genre of games are released the most ?
  6. What genre of games are rated the highest?
  7. How does certain scoring on games vary across different genres ?
  8. What keywords are usually associated with video games ?
  9. How do these keywords differ by genre ?
  10. Are PC game requirements more demanding now than before?
As this project was an exercise to explore python and its visualization capabilities, there were a few additional goals for the project:
  • The graphs would be made entirely using python libraries but pushing beyond plotly and matplotlib
  • Aesthetically pleasing through color scheme that is both functional and accessible to broader audience
  • The graphs should be highly interactive and dynamic

Approach

The very first hurdle in building this project was the data. Since the input data came from two different sources, a lot of data processing went behind the scenes to combine information from both the sources and making sure there are no missing values in the correct columns. Since the project was built in python after the intial data compilation into a single file the rest of the process was fast and scalable. The interesting aspect of the project was that all 4 topics being studied had the same source but had completely different levels of data being displayed via graphs. The first task was to combine player count csv files into one single file. This was a huge endeavor given the large amount of files for each video game. In order to keep data at a manageable level only the player counts at start of each month were retained. The graph was always going to be time series graph but as there were over 12k games it would become extremely cluttered if all games were displayed. Therefore a list of 20 popular games were selected in the year 2017 along with the functionality to select and study each game individually. The list conatined a wide varity of singleplayer and multiplayer games as we wanted to compare both kinds of games. The graph was given additional functionality to view just one, a few or all the games for comparison, a zoom feature as well as a reset feature to restore the graph to original form.

Even for subsequent charts such as assigning a genre to each game required data transformations to assign each game with the most optimal genre. Similarily, for the game descriptions I had to process the associated text combined from all games and remove gibberish as well as english stop words. Once we removed all the common english words we could find words that are actually relevant and sometimes even unique to a particular genre of games. Once all intial datasets were finalized, I started designing the visuals keeping interactivity in mind and connecting information spread across multiple graphs.

I tried to replicate the results using the color templates I had used in project 1 but unfortunately since the number of data points and levels was too large it became a challenge to keep track of each entity. In the end I went ahead with the predefined set of color combinations provided by Altair but made sure all colors blend well with the theme and background of the app while still being distinct enough to differentiate categories from one other.

As this project was mostly an exploratory exercise, the entire project was a lot of fun to test out in python. Exploring libraries was both challenging and fun but Altair proved to be an extremely powerful tool into making interactive visualizations. The most amount of work went into experimenting with libraries and their capabilities and what sort of interactivity made the most sense. Despite the newer versions of libraries there was documentation available which was helpful. Overall I'm realy happy and proud of the kind of interactivity I was able to achive and how useful the graphs are to deepdive into the data. I enjoyed using Altair and would definetely recommend using it for data visualization.

Observations

  1. Player Retention
    • Using the timeline gragh, we can see that multiplayer games such as CS:GO and Dota 2 fetch a larger player base as compared to single player games such as Witcher 3.
    • Free to play games such as Path of exile and Warframe have fluctuating player base and hugely rely on seasonal content releases to bring players back to play the game
    • Multiplayer games that are built on hype can also loose a lot of player base if the games do not deliver on new content such as: Player Unknown Battlegrounds
    • Game decisions such as going from a paid version to "free to play" or having a tie-up with a Netflix TV series "The Witcher" can boost player base significantly as observed in Rocket League and The Witcher 3
  2. Video Game Scores
    • Given the massive size of the industry that is continously growing there are more video games being released in recent years as compared to older times
    • Multiplayer games are often rated lower as it is trickier to find a balance between tweaking the power of the video game elements
    • Action games and Strategy games are the most common Genre of game being released
    • RPG and strategy games are often rated highter than other games not by a huge margin.
  3. Video Game Description
    • Certain keywords are common across all genre of games and are used very frequently such as "New","Game","World","Players","Experience"
    • Some genres of games can have their own set of unique words which make the most sense for the kind of games being described such as:
      • Action : Weapons, Action
      • Adventure: Explore, World, Puzzles
      • Strategy: Simulation, Civilization, Map, Build
      • Multiplayer: Combat, squad, Online
      • RPG: Character, Experience, Fantasy, Explore, Skills
      • Sports: Driving, Realistic, Cars , Race
  4. Video Game Requirements
    • Although the sparsity of data in intial years, games released before 2010 512 MB of RAM memory was the most suggested PC specification
    • Post 2010 we see 4 GB being more recommended whereas 512 MB is one of the least recommneded specification
